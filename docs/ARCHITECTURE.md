# NYOTA — EXPERIENCE ARCHITECTURE

## Core Concept

Nyota is not a traditional website.

It is an **Interactive Editorial Universe** combining:

* Storytelling (Destinations)
* Editorial Authority (Journals)
* Exploration (Universe Map)
* Philosophy (Nyota Lens)
* Conversion (Discovery Briefing)

---

## Primary User Journey

HOME
→ UNIVERSE MAP
→ STORY
→ JOURNAL
→ PHILOSOPHY
→ DISCOVERY BRIEFING

This flow must always be preserved.

---

## Core Sections

### 1. Universe

* Interactive constellation map
* Powered by story JSON (`mapCoords`)
* Primary navigation system

### 2. Stories (Destinations)

Each story is:

* Cinematic
* Structured
* Component-driven

Layout:

* Hero
* Nyota Lens
* Chapters (media-driven)
* Pairing (next story)

---

### 3. Journals (Editorial Layer)

Purpose:

* Build authority
* Educate high-value clients

Style:

* Minimal
* Serif-heavy
* Magazine-like

---

### 4. Philosophy (Nyota Lens)

* Defines brand thinking
* Reinforces positioning

---

### 5. Discovery Briefing

* Conversion point
* Should feel like concierge intake, not a form

---

## Design Principles

* Cinematic > Functional
* Editorial > Commercial
* Minimal > Dense
* Narrative > Navigation

---

## Core Experience Loop

MAP → STORY → JOURNAL → INQUIRY

This loop is the foundation of the entire product.
